## Installation

### Quick Install (npm)
```bash
npm install -g clawdbot@latest
```

### From Source (development)
```bash
git clone https://github.com/clawdbot/clawdbot.git
cd clawdbot
pnpm install
pnpm ui:build
pnpm build
```

### Onboarding
```bash
clawdbot onboard --install-daemon
```

## Key Paths

| Path | Purpose |
|------|---------|
| `~/.clawdbot/clawdbot.json` | Gateway config |
| `~/.clawdbot/agents/` | Agent configs + sessions |
| `~/.clawdbot/credentials/` | OAuth credentials |
| `~/clawd/` | Workspace (skills, prompts, memories) |
| `/tmp/clawdbot/` | Logs |

## Runtime Notes

- **Node only** — Bun is NOT recommended (known issues with WhatsApp/Telegram)
- **One Gateway per host** — Multiple gateways need isolated profiles/ports
- **Loopback-first** — Gateway WS defaults to `ws://127.0.0.1:18789`
- **Gateway token** — Generated by default (even on loopback), stored in `gateway.auth.token`
- **Linux**: Enable lingering for systemd user services: `sudo loginctl enable-linger $USER`

## Update Without Breaking
- Keep `~/clawd` and `~/.clawdbot/` as personal config
- Don't put personal prompts into the clawdbot repo
- Update: `npm install -g clawdbot@latest` then `clawdbot doctor`
